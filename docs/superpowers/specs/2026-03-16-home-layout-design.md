# 홈 페이지 카드형 미니 대시보드 설계

## 목표

- 홈의 4개 섹션 카드가 각각 독립적인 미니 대시보드로 동작해야 한다.
- 섹션은 `코드 자산 / 지적 자산 / 트러블슈팅 / Lesson & Learned`로 고정한다.
- 각 카드 안에서 해당 섹션 전용 지표를 바로 보고 섹션 목록으로 이동할 수 있어야 한다.

## 카드별 지표

- 공통
  - 총 자산 수
  - 새 글 증감
  - 새 댓글 증감
- 관리자 전용
  - 승인 대기 자산 수

## 집계 기준

- 총 자산 수: 해당 kind의 `approved` 자산 수
- 새 글 증감: 오늘 생성된 `approved` 자산 수 - 어제 생성된 `approved` 자산 수
- 새 댓글 증감: 해당 kind 자산에 달린 오늘 생성 댓글 수 - 어제 생성 댓글 수
- 승인 대기 자산 수: 해당 kind의 `pending` 자산 수
- 날짜 경계는 KST 기준 하루 단위로 계산한다.

## 표시 규칙

- 관리자는 카드당 4개 KPI를 1행 rail로 본다.
- 비관리자는 카드당 3개 KPI를 1행 rail로 본다.
- KPI rail은 모든 화면에서 줄바꿈하지 않는다.
- 카드 폭이 부족한 화면에서는 KPI rail만 카드 내부에서 가로 스크롤된다.
- 증감값은 부호 포함 문자열(`+3`, `-1`, `0`)로 표시한다.

## 비목표

- 게시판(Q&A/공지) 댓글을 섹션 카드 KPI에 포함하지 않는다.
- 상단 공통 KPI 스트립은 두지 않는다.
- 공용 히어로 컴포넌트는 수정하지 않는다.
