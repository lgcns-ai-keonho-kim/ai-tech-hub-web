# One Monokai Dark System

## 1. 문서 목표

이 문서는 현재의 Instagram 레이어형 다크 UI를 **One Monokai 색 체계**로 재정렬하기 위한 기준 문서다.  
핵심 방향은 `Instagram의 장면 리듬 50%`와 `One Monokai의 정제된 코드 에디터 감성 50%`의 결합이다.

이 문서는 아래 항목을 매 루프마다 다시 논쟁하지 않도록 고정한다.

- 어떤 색이 제품의 기본 언어가 되는지
- 어떤 색이 상태, 경고, 강조를 맡는지
- 배경 오브와 표면 광량이 어떤 톤으로 움직여야 하는지
- 로고 자산을 재색상화하지 않고 어떻게 장면에 통합할지
- 카탈로그, 상세, Lessons, Admin이 같은 시스템 안에서 어떻게 다른 감정 톤을 가져야 하는지

## 2. 제품의 시각적 정체성

### 기본 정의

- 이 제품은 소셜 피드가 아니라 `탐색형 AI 허브`다.
- Instagram에서 차용하는 요소는 레이아웃 복제가 아니라 공기감, 부유감, 촉각적 표면, 부드러운 모션 리듬이다.
- 최종 컬러 체계는 One Monokai를 기준으로 고정한다.
- 즉, 장면은 감성적이어야 하지만 색 의미는 코드 에디터처럼 명확해야 한다.

### 절대 유지 원칙

- 다크 테마만 사용한다.
- 배경은 단일 검정이 아니라 깊이 있는 **에디터형 캔버스**처럼 보여야 한다.
- 기본 CTA와 활성 상태는 **블루(`#61AFEF`)**가 맡는다.
- 보라(`#C678DD`)는 보조 강조, 선택된 상호작용, 주변광에 사용한다.
- 그린(`#98C379`)은 성공, 유효, 긍정 신호에만 사용한다.
- 골드(`#E5C07B`)는 주의, 지표 포인트, 생산성 화면의 따뜻한 보조 강조에 사용한다.
- 핑크 레드(`#E06C75`)는 오류, 위험, 파괴적 액션에만 사용한다.
- 로고 이미지는 원본 유지다. 제품 팔레트에 맞춘 주변광만 허용한다.

### 감성 키워드

- 에디터 같은 질서감
- 부유하는 블루와 보라 광
- 정제된 생산성
- 촉각적인 표면
- 조용하지만 살아 있는 콘솔
- 기술 문서와 탐색 카탈로그의 혼합

## 3. 전역 비주얼 원칙

### 3.1 색 체계

One Monokai 공식 팔레트를 기준으로 아래 역할을 고정한다.

- 기본 바탕색
  - 메인 캔버스: `#282C34`
  - 더 깊은 보조 배경: `#1F232A`에 가까운 파생 단계
  - 카드 바탕: 메인 캔버스보다 4~8% 밝은 단계
- 기본 텍스트
  - 주 텍스트: `#ABB2BF`
  - 약한 텍스트: `#7F8794` ~ `#8E97A6` 범위 파생 단계
- 핵심 액션색
  - 기본 CTA, 활성 탭, 포커스 링, 주요 링크: `#61AFEF`
- 보조 강조색
  - 선택된 필터 배경, 히어로 주변광, 보조 액션 강조: `#C678DD`
- 상태색
  - 성공: `#98C379`
  - 주의/메트릭 포인트: `#E5C07B`
  - 오류/위험: `#E06C75`
  - 웜 보조색: `#D19A66`

### 3.2 배경 시스템

배경은 아래 3개 층을 동시에 가진다.

- 베이스 레이어
  - One Monokai 배경색에 가까운 잉크 캔버스
  - 순수 검정 대신 아주 약한 블루-그레이를 포함해야 한다
- 무드 레이어
  - 대형 블러 오브 2~3개를 배치한다
  - 블루 오브가 주도권을 가지며, 보라와 골드는 장면 보조로만 사용한다
  - 핑크 레드는 배경 주연이 아니라 위험 상태 또는 일부 극소량의 대비 포인트로만 사용한다
- 질감 레이어
  - 미세한 그레인
  - 저대비 격자 또는 선형 패턴
  - 코드 에디터의 정돈감을 암시하는 수준에서만 사용한다

### 3.3 입체감 시스템

입체감은 `바닥면 + 표면 + 상단 하이라이트 + 내부 광량 차이`의 조합으로 만든다.

- 패널은 배경에서 1단계 떠 있어야 한다
- hover 시 카드가 들리더라도 과한 3D 장난감처럼 보여서는 안 된다
- 블루와 보라의 경계광은 얇고 정제되어야 한다
- Admin 화면은 카탈로그보다 광량이 약하고 더 구조적이어야 한다

### 3.4 타이포그래피

- 기본 UI 폰트는 `Figtree`를 유지한다
- 헤드라인은 조금 더 단단하고 압축된 인상을 가져야 한다
- 섹션 캡션은 편집적이되 코드 에디터의 라벨처럼 너무 기계적이면 안 된다
- 본문은 65~75자 내외의 읽기 폭을 유지한다
- 숫자 메트릭은 라벨보다 선명하고 고립된 위치를 가져야 한다

## 4. 모션 시스템

### 4.1 모션 기조

모션 밀도는 높게 유지하지만 과장된 장식이 아니라 **조작감과 공기감**을 형성해야 한다.

- 마이크로 인터랙션: 140ms ~ 220ms
- 카드 hover 및 상태 변화: 180ms ~ 260ms
- 패널 진입 및 탭 전환: 240ms ~ 360ms
- 배경 오브 이동: 8초 ~ 20초
- 페이지 진입 스테이저: 320ms ~ 560ms

### 4.2 모션 규칙

- 모든 움직임은 `transform`과 `opacity` 중심으로 설계한다
- hover는 `lift + border glow + 내부 광량 변화`를 함께 사용한다
- 버튼은 눌릴 때 짧은 압축감을 준다
- 탭 전환은 하드컷을 피하고 부드러운 페이드와 미세한 이동을 사용한다
- skeleton과 로딩 shimmer는 대각선 또는 측면 흐름을 사용하되 저채도로 유지한다

### 4.3 Reduced Motion

`prefers-reduced-motion` 환경에서는 아래를 적용한다.

- 배경 오브 이동 중지
- hover 기울기 제거
- 페이지 진입 스테이저 제거
- opacity 중심 상태 변화만 유지

## 5. 상호작용 원칙

- 상호작용 가능한 요소는 정적 상태와 hover 상태가 분명히 구분되어야 한다
- 포커스 링은 블루를 기본으로 하되 아주 약한 보라 후광을 허용한다
- 드롭다운과 다이얼로그는 배경과 분리된 부유층처럼 보여야 한다
- 검색 입력과 작성 입력은 단순 폼 필드가 아니라 작업 표면의 중심 도구처럼 보여야 한다
- 비활성 상태는 명도와 채도를 모두 낮춰 존재감을 줄인다
- 오류는 핑크 레드로 명확히 드러내되 배경 전체를 붉게 물들이지 않는다

## 6. 로고 자산 활용 원칙

### `assets/logo/logo.png`

- 헤더 좌측 공식 워드마크로 사용한다
- 홈 히어로, 로그인 모달, Empty 상태의 브랜드 앵커로 사용할 수 있다
- 로고 자체는 재색상화하지 않는다
- 로고 뒤에는 블루 또는 보라 계열의 매우 약한 주변광만 허용한다

### `assets/logo/farvicon.png`

- 정사각 심볼 자산으로 취급한다
- 모바일 헤더, 작은 상태 심볼, 로딩 상태, 배경의 희소한 시그널 오브젝트로 사용한다
- 반복 패턴처럼 남발하지 않고 작은 구간에서만 전략적으로 사용한다

## 7. 화면 공통 구성 언어

모든 화면은 아래 다섯 레이어를 공유해야 한다.

- 레이어 1: One Monokai 전역 배경 무드
- 레이어 2: 상단 글로벌 레일
- 레이어 3: 페이지 히어로 또는 맥락 패널
- 레이어 4: 실제 작업 영역
- 레이어 5: 상태 피드백, 다이얼로그, 드롭다운

## 8. 화면군별 감정 톤 차등

- 카탈로그
  - 블루가 가장 강하고 보라가 보조로 흐르는 탐색 무드
- 상세
  - 블루 기반 집중감에 카테고리별 보라/골드 차등
- Lessons
  - 골드와 블루가 균형을 이루는 생산적 지식 공간
- Admin
  - 보라와 블루 중심의 정돈된 운영 콘솔

## 9. 금지 사항

- 기본 CTA를 여러 색으로 분산하는 것 금지
- 레드나 핑크를 배경 주연으로 사용하는 것 금지
- 밝은 배경 블록 도입 금지
- 과도한 유리 질감과 강한 글래스모피즘 금지
- 의미 없는 반복 애니메이션 금지
- One Monokai를 표면 장식으로만 쓰고 상태 의미를 흐리는 것 금지

## 10. 구현 우선순위

1. 전역 색 토큰과 배경 오브를 One Monokai로 전환
2. CTA, 탭, 포커스, 상태 패널의 의미 색 재정의
3. 홈 카탈로그와 카드의 색 계층 재설계
4. 상세 마크다운, Mermaid, Snapshot 표면 재정의
5. Lessons와 Admin의 생산성 톤 정리

## 11. 완료 기준

이 문서를 기준으로 후속 구현자가 아래를 추가 질문 없이 결정할 수 있어야 한다.

- One Monokai 색의 역할 분담
- CTA와 보조 강조의 사용 범위
- 배경 오브와 경계광의 주도 색
- 로고의 사용 위치와 금지 규칙
- 각 화면군의 감정 톤 차이
- Reduced Motion 시 동작
