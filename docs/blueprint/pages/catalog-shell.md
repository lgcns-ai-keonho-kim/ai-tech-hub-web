> 목적: 카탈로그와 전역 셸 UI를 One Monokai 기반 다크 톤으로 재설계하기 위한 화면 규칙을 정의한다.
> 설명: 헤더, 홈 히어로, 검색/필터, 사이드바, 카드 그리드, 모바일 대응까지 실제 구현 수준으로 기준을 고정한다.
> 적용 패턴: 화면군 블루프린트 패턴
> 참조: `ui/src/widgets/global-header.tsx`, `ui/src/pages/home-page.tsx`, `ui/src/widgets/agent-card.tsx`

# Catalog Shell Blueprint

## 1. 대상 화면

- `/`
- 공용 헤더
- 전역 에이전트 선택기
- 홈 히어로
- 카테고리 사이드바
- 검색, 정렬, 모바일 필터 탭
- 에이전트 카드 그리드

## 2. 목표 인상

카탈로그 첫 화면은 “정리된 기술 대시보드”보다 “살아 있는 프리미엄 탐색 공간”처럼 보여야 한다.  
사용자는 들어오자마자 아래 감정을 느껴야 한다.

- 이 제품은 활력이 있다.
- AI 자산이 정적 목록이 아니라 움직이는 생태계처럼 보인다.
- 화면이 화려하지만 산만하지 않다.
- 무엇을 눌러야 하는지 명확하다.

## 3. 헤더 규칙

- 헤더는 단순 고정 바가 아니라 부유하는 글로벌 레일처럼 보여야 한다.
- 기본 형태는 매우 어두운 반투명 패널 위에 얇은 블루 하이라이트 라인이 있는 구조로 잡는다.
- `logo.png` 워드마크를 좌측 브랜드 앵커로 사용한다.
- 메뉴 활성 상태는 사각 탭보다 `블루 저면광 + 얇은 활성 바`가 더 잘 어울린다.
- 글로벌 에이전트 선택기는 일반 Select보다 `시그널 컨트롤`처럼 보여야 한다.
- 로그인 버튼은 기본 CTA보다 살짝 더 감성적이어도 되지만, 제품 전체를 흔드는 정도로 화려하면 안 된다.
- 모바일에서는 `farvicon.png` 심볼을 먼저 보이고, 워드마크는 축소하거나 숨긴다.

## 4. 홈 히어로 규칙

- 현재의 단순 2열 패널을 유지하되 시각적 위계를 강하게 바꾼다.
- 좌측은 제품 소개보다 `탐색 욕구를 자극하는 메인 스테이지`처럼 보이게 한다.
- 배경에는 대형 블루 오브와 약한 보라 오브, 아주 절제된 골드 보조광을 둔다.
- 타이틀은 현재보다 더 크게, 더 짧게, 더 확신 있게 보이게 한다.
- 서브카피는 기술 데모 소개문이 아니라 사용자가 무엇을 할 수 있는지 중심으로 재작성될 수 있도록 공간감을 준다.
- 우측 글로벌 에이전트 패널은 `선택된 AI 페르소나`를 보여주는 하이라이트 카드처럼 다룬다.
- 히어로 전체는 첫 진입 시 아래에서 위로 등장하는 대신, `미세한 확대 축소와 페이드`로 고급스럽게 진입한다.

## 5. 검색과 필터 바

- 검색 입력은 단순 툴바가 아니라 메인 탐색 인터페이스다.
- 좌측 검색 입력은 가장 큰 시각적 비중을 가져야 한다.
- 초기화 버튼은 회색 아웃라인 버튼으로 두되 hover 시 배경과 경계광이 동시에 반응하도록 한다.
- 정렬 Select는 독립 컨트롤이 아니라 필터 바 안의 일원처럼 보이게 통일한다.
- 모바일 카테고리 탭은 블루 활성 상태가 분명한 필름형 탭이 적합하다.
- 포커스 상태에서는 입력 주변광이 과하지 않게 퍼져야 한다.

## 6. 카테고리 사이드바

- 데스크톱 사이드바는 기능적 목록을 넘어서 `탐색 레이더`처럼 느껴져야 한다.
- 카테고리 리스트 항목은 hover 시 텍스트만 바뀌지 말고 배경 아래쪽에서 블루 또는 보라 에지가 올라오게 한다.
- 활성 카테고리는 보더 강조보다 `은은한 블루 필`과 앞쪽 정렬감으로 표현한다.
- 사이드바 상단 라벨은 기술 라벨 느낌보다 편집적 캡션 감성으로 조정한다.

## 7. 카드 그리드 규칙

- 카드 그리드는 현재와 같은 균일 그리드를 유지하되 카드 자체가 더 풍부해야 한다.
- 모든 카드에 썸네일을 강제하지 않는다.
- 대신 카드 내부를 다음 4구역으로 분명히 나눈다.
  - 카테고리 및 날짜
  - 제목과 요약
  - 메트릭 구획
  - 하단 액션 구획
- 메트릭 칩은 현재보다 더 작고 촘촘하지만, 숫자는 더 두드러져야 한다.
- 카드 hover는 아래를 동시에 수행한다.
  - 카드가 4px~8px 정도 떠오름
  - 상단 블루-보라 경계광이 조금 밝아짐
  - 배경 내부 오브가 천천히 움직이는 듯한 광량 변화
  - GitHub 버튼 대비가 상승
- 카드 간 시각적 단조로움을 막기 위해 목록 내에서 일부 카드는 보라 또는 골드 보조 무드를 허용한다.

## 8. 빈 상태와 오류 상태

- Empty는 단순 dashed border 박스 대신 브랜드 톤이 있는 정적 무드 패널이어야 한다.
- `farvicon.png`를 작은 심볼로 배치할 수 있다.
- 오류 Alert는 현재처럼 강한 빨강 경고만 쓰지 말고, 배경은 어둡게 유지하면서 경계와 아이콘에만 핑크 레드 경고 색을 집중한다.

## 9. 모바일 규칙

- 모바일에서는 헤더, 히어로, 검색, 카드 사이 간격이 너무 넓어 보이지 않도록 리듬을 다시 잡는다.
- 히어로는 텍스트와 하이라이트 카드가 겹친 듯한 깊이를 가져도 된다.
- 카드 hover가 없는 환경에서도 눌림 감각이 느껴지도록 pressed 상태 규칙을 별도로 둔다.
- 탭과 버튼은 최소 44px 터치 타깃을 유지한다.

## 10. 구현 시 우선순위

1. 헤더에 실제 로고 자산 적용
2. 전역 배경의 움직이는 오브 레이어 도입
3. 홈 히어로 재구성
4. 필터 바 감성 개선
5. 카드 깊이와 hover 시스템 교체
6. 모바일 탭과 카드 pressed 상태 정리
