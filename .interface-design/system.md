# 사이드바 레이아웃 시스템

## 방향과 분위기
- 보호 영역은 상단 헤더 대신 좌측 수직 사이드바 셸을 기본 구조로 사용한다.
- 전체 인상은 조용하고 밀도 높은 업무용 도구처럼 유지한다.
- 정보 위계는 강한 색 대비보다 배치, 여백, 상태 변화로 드러낸다.

## 깊이 전략
- 기본 전략은 `borders-only + subtle surface shift` 조합이다.
- 사이드바와 메인 콘텐츠는 같은 배경 온도를 유지하고 얇은 경계선으로만 분리한다.
- 상호작용 버튼은 테두리를 과하게 드러내지 않고 hover/background 변화로 반응한다.

## 간격 기준
- 기본 간격 단위는 4px 배수로 맞춘다.
- 사이드바 내부는 `gap-3`, 주요 섹션 간은 `gap-4`를 기준으로 쓴다.
- 메인 콘텐츠 셸은 데스크톱에서 `px-6~8`, 모바일에서 `px-4`를 기준으로 유지한다.

## 핵심 패턴
- 사이드바는 펼침/접힘을 지원하며 전환 시간은 `300ms`, easing은 `cubic-bezier(0.22, 1, 0.36, 1)`을 사용한다.
- 접힘 상태에서는 브랜드 로고를 심볼만 노출하고, 메뉴는 아이콘만 남긴다.
- 펼침 상태에서는 로고 워드마크, 메뉴 라벨, 알림 라벨, 유저 이름이 함께 드러난다.
- 알림과 유저 버튼은 사이드바 상단 조작군에 배치하고, 테마 토글은 메인 콘텐츠 상단 우측에 유지한다.
- 유저 메뉴는 클릭보다 hover/focus 진입을 우선하고, 트리거와 메뉴 사이에는 hover-safe bridge를 둔다.
- 유저 메뉴 닫힘은 즉시 닫지 않고 `180ms` 지연 후 처리해 대각선 이동에서도 메뉴가 유지되게 한다.
- 사이드바 hover 확장과 유저 메뉴 open 상태는 분리하되, 유저 메뉴가 열린 동안은 사이드바가 접히지 않게 유지한다.
